# Internationalization (i18n) Guide

## Overview

PicStitch supports multiple languages with a complete internationalization system. The default language is English, and users can switch between languages using the language selector in the navigation bar.

## Supported Languages

- **English (en)** - Default language
- **简体中文 (zh)** - Simplified Chinese

## How It Works

### Language Storage
- User's language preference is stored in `localStorage` with the key `picstitch_lang`
- The language persists across browser sessions
- Default language is English if no preference is stored

### Language Switching
- Click the language selector (🌐) in the top navigation bar
- The interface will instantly update to the selected language
- All text, including templates, buttons, and labels, will be translated

## Adding New Languages

To add a new language:

1. Open `src/i18n.js`
2. Add a new language object to the `translations` object
3. Follow the same structure as existing languages
4. Use the language code (e.g., 'fr' for French, 'es' for Spanish)

Example:
```javascript
fr: {
    nav: {
        home: 'Accueil',
        features: 'Fonctionnalités',
        // ... more translations
    },
    // ... rest of translations
}
```

5. Update the language switcher logic in `src/main.js` if needed

## Translation Keys Structure

All translations are organized hierarchically:

- `nav.*` - Navigation items
- `header.*` - Header section (title, subtitle, badges, description)
- `sidebar.*` - Sidebar controls and labels
- `templates.*` - Template names
- `imagePanel.*` - Image panel labels
- `canvas.*` - Canvas placeholder text
- `export.*` - Export-related messages

## Dynamic Content

Some translations support parameters:
- `sidebar.imagesLoaded: '{count} images loaded'` - The `{count}` will be replaced with the actual number

## Best Practices

1. **Always use translation keys** - Never hardcode text in HTML or JavaScript
2. **Keep keys descriptive** - Use clear, hierarchical key names
3. **Test both languages** - Ensure all UI elements are properly translated
4. **Maintain consistency** - Use consistent terminology across translations

## Technical Details

- The i18n system uses a singleton pattern (`i18n` instance)
- Translations are loaded immediately when the class is instantiated
- Language changes trigger a `language-changed` event
- All elements with `data-i18n` attribute are automatically updated
