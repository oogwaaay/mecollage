## 🔍 如何查看详细日志

### 步骤 1：查看当前运行状态

1. **点击正在运行的工作流**（#4）
2. **展开 "build-and-deploy" job**
3. **查看每个步骤的日志**：
   - 哪个步骤正在运行？
   - 是否有错误信息？
   - 是否有警告？

### 步骤 2：检查卡住的步骤

常见卡住的步骤：

**如果是 "Install dependencies" 卡住**：
- 可能是某个 npm 包下载慢
- 可能是网络问题

**如果是 "Build" 卡住**：
- 可能是构建过程有问题
- 可能是内存不足

**如果是 "Deploy to GitHub Pages" 卡住**：
- 可能是推送大文件慢
- 可能是 GitHub API 限流

## 🛠️ 解决方案

### 方案 1：取消并重新运行

1. **点击正在运行的工作流**
2. **点击右上角的 "Cancel workflow"**
3. **等待取消完成**
4. **点击 "Re-run all jobs"** 重新运行

### 方案 2：检查并修复问题

如果工作流失败，查看错误日志：

1. **点击失败的工作流**
2. **展开失败的步骤**
3. **查看错误信息**
4. **根据错误信息修复问题**

### 方案 3：优化构建配置

如果构建时间过长，可以：

1. **减少构建输出**：检查是否有不必要的文件
2. **优化依赖**：检查 `package.json` 是否有不必要的依赖
3. **使用缓存**：确保 npm 缓存正常工作

## 🚨 如果工作流失败

### 常见错误及解决：

**错误：npm ci 失败**
- 解决：检查 `package-lock.json` 是否最新
- 解决：确保所有依赖版本兼容

**错误：Build 失败**
- 解决：检查构建日志中的具体错误
- 解决：确保所有文件路径正确

**错误：Deploy 失败**
- 解决：检查 GitHub Pages 是否已启用
- 解决：检查权限设置是否正确

## 💡 建议

1. **先查看详细日志**：了解具体卡在哪一步
2. **如果超过 10 分钟**：会自动超时失败，然后可以重新运行
3. **如果持续失败**：可能需要检查代码或配置问题
