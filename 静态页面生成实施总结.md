## 使用方法

### 开发环境
```bash
npm run dev  # 正常开发，不生成静态页面
```

### 构建（包含静态页面）
```bash
npm run build:static
```

### 单独提取博客数据
```bash
npm run extract-blog-data
```

### 单独生成静态页面（需要先构建）
```bash
npm run build
npm run generate-static
```

### 部署
```bash
npm run deploy  # 自动执行 build:static 然后部署
```

## 工作原理

1. **数据提取阶段**
   - 运行 `extract-blog-data.js`
   - 从 `src/blog-manager.js` 导入博客数据
   - 保存为 `scripts/blog-data.json`

2. **构建阶段**
   - 运行 `vite build`
   - 生成 `dist/index.html` 和所有资源文件

3. **静态页面生成阶段**
   - 运行 `generate-static-pages.js`
   - 读取 `dist/index.html` 作为模板
   - 读取 `scripts/blog-data.json` 获取博客数据
   - 为每篇博客文章生成 3 个语言版本的 HTML
   - 为 features 和 tutorial 生成静态 HTML
   - 更新每个页面的 SEO meta 标签
   - 设置正确的页面显示/隐藏状态

## SEO 优化效果

### 解决的问题
1. ✅ **前端渲染问题（部分解决）**
   - 博客文章、功能页、教程页现在都是静态 HTML
   - 搜索引擎可以直接抓取内容
   - 无需等待 JavaScript 执行

2. ✅ **真实 URL 支持**
   - 每个页面都有独立的 URL
   - 可以直接访问 `/blog/how-to-make-photo-collage.html`
   - 支持多语言版本

3. ✅ **独立的 SEO meta 标签**
   - 每个页面都有针对性的 title、description
   - 每个页面都有正确的 canonical URL
   - OG tags 和 Twitter Card 都已优化

### 仍保持客户端渲染的部分
- 首页（拼图编辑器）- 需要交互功能
- 博客列表页 - 需要搜索和筛选功能
- 所有动态功能 - 保持原有体验

## 注意事项

1. **资源路径**
   - 生成的静态 HTML 使用相对路径引用资源
   - 确保所有资源文件都在正确的位置

2. **多语言支持**
   - 英文版本：`blog/post-id.html`
   - 中文版本：`blog/post-id.zh.html`
   - 西班牙语版本：`blog/post-id.es.html`
   - 路由会自动处理语言后缀

3. **内容更新**
   - 修改博客内容后，需要重新运行 `build:static`
   - 新增博客文章后，会自动生成对应的静态页面

4. **GitHub Pages 部署**
   - 静态 HTML 文件会正常部署
   - 确保 `.gitignore` 不忽略 `dist/` 目录
