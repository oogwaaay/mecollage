## 📤 方式一：自动部署（推荐，最简单）✨

### 第一次设置（只需一次）：

1. **在 GitHub 上启用 GitHub Actions**：
   - 进入您的 GitHub 仓库
   - 点击 **Settings** → **Pages**
   - 在 **Source** 中选择：**GitHub Actions**
   - 保存

2. **提交所有更改到 GitHub**：
   ```bash
   git add .
   git commit -m "Add SEO optimizations and auto-deployment"
   git push
   ```

3. **等待自动部署**：
   - GitHub Actions 会自动运行（约 2-3 分钟）
   - 在仓库页面点击 **Actions** 标签可以查看进度
   - 完成后网站自动更新！

### 以后每次更新：

**超级简单！** 只需要：

```bash
# 1. 修改代码后
git add .
git commit -m "更新说明"
git push
```

然后等待 2-3 分钟，网站就会自动更新！✨

## 📤 方式二：手动部署（如果不想用自动部署）

### 步骤 1：安装 gh-pages

```bash
npm install --save-dev gh-pages
```

### 步骤 2：运行部署命令

```bash
npm run deploy
```

这个命令会：
1. 自动构建项目（`npm run build`）
2. 自动将 `dist` 目录推送到 `gh-pages` 分支
3. GitHub Pages 会自动更新网站

## ⚠️ 重要提示

1. **GitHub Pages 不支持 .htaccess**：
   - `.htaccess` 文件只在 Apache 服务器生效
   - GitHub Pages 使用 Nginx，会自动处理缓存和压缩
   - 所以 `.htaccess` 文件可以保留，但不会生效（不影响功能）

2. **robots.txt 会正常部署**：
   - `public/robots.txt` 会自动复制到 `dist/robots.txt`
   - 部署后可以通过 `https://www.mecollage.top/robots.txt` 访问

3. **自动部署的优势**：
   - 每次 `git push` 后自动更新
   - 不需要手动构建和上传
   - 部署历史可追溯（在 Actions 标签中查看）

## 🆘 遇到问题？

### GitHub Actions 没有运行？
- 检查 Settings → Actions → General
- 确保 "Workflow permissions" 设置为 "Read and write permissions"

### 部署失败？
- 查看 Actions 标签中的错误信息
- 检查构建日志，找出问题所在

### 网站没有更新？
- 等待 2-3 分钟
- 清除浏览器缓存
- 检查 GitHub Pages 设置是否正确
