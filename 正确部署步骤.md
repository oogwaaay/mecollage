# ✅ 正确的部署步骤（已验证方案）

## 🔍 问题分析

根据错误信息：
- "HttpError: Not Found"
- "Get Pages site failed"

这说明 GitHub Pages 可能没有正确启用，或者使用了不兼容的部署方式。

## ✅ 解决方案

我已经将部署配置改为使用 `peaceiris/actions-gh-pages@v3`，这是：
- ✅ 最稳定、最广泛使用的 GitHub Pages 部署方案
- ✅ 不需要复杂的 environment 配置
- ✅ 适用于所有类型的仓库（公开/私有）

## 📋 必须完成的步骤

### 步骤 1：在 GitHub 上启用 Pages（重要！）

1. **打开您的 GitHub 仓库**
2. **点击 Settings**（仓库设置）
3. **点击左侧的 Pages**
4. **在 "Source" 部分**：
   - 如果显示 "Deploy from a branch"，**先不要改**
   - 如果显示 "GitHub Actions"，保持即可
5. **确保 "Custom domain" 设置为**：`www.mecollage.top`（如果有）
6. **点击 Save**

### 步骤 2：检查 Actions 权限

1. **在 Settings 中，点击 Actions** → **General**
2. **滚动到 "Workflow permissions"**
3. **选择 "Read and write permissions"**
4. **点击 Save**

### 步骤 3：提交修复后的配置

在 GitHub Desktop 中：

1. **查看更改**：`.github/workflows/deploy.yml` 文件已更新
2. **输入提交信息**：
   ```
   Use peaceiris/actions-gh-pages for reliable deployment
   ```
3. **点击 "Commit to main"**
4. **点击 "Push origin"**

### 步骤 4：等待部署完成

1. **进入 Actions 标签**
2. **查看新的工作流运行**
3. **等待完成**（约 1-2 分钟）

## 🔍 如果仍然失败

### 检查清单：

1. ✅ **GitHub Pages 已启用**（Settings → Pages）
2. ✅ **Actions 权限正确**（Settings → Actions → General → Read and write）
3. ✅ **工作流文件已推送**（`.github/workflows/deploy.yml`）
4. ✅ **package.json 中有 build 脚本**

### 查看详细错误：

1. **点击失败的运行记录**
2. **展开 "build-and-deploy" job**
3. **查看每个步骤的日志**
4. **找到红色 X 标记的步骤**
5. **查看具体错误信息**

### 常见错误及解决：

**错误：Permission denied**
- 解决：Settings → Actions → General → Workflow permissions → Read and write

**错误：Build failed**
- 解决：检查构建日志，可能是依赖问题

**错误：Deploy failed - 404**
- 解决：确保 GitHub Pages 已启用（Settings → Pages）
