# 🔧 修复 GitHub Actions 部署失败

## 问题原因

GitHub Actions 部署失败通常是因为：
1. **缺少权限配置** - 工作流需要明确的权限设置
2. **使用了旧版部署方式** - 需要使用新的 GitHub Pages Actions

## ✅ 已修复

我已经更新了 `.github/workflows/deploy.yml` 文件，修复了以下问题：

1. ✅ 添加了 `permissions` 配置（允许写入 Pages）
2. ✅ 添加了 `concurrency` 配置（避免并发部署冲突）
3. ✅ 更新为使用官方的 GitHub Pages Actions（`actions/deploy-pages@v4`）
4. ✅ 添加了 `environment` 配置

## 📤 下一步操作

### 1. 提交修复后的配置

在 GitHub Desktop 中：

1. **查看更改**：应该会看到 `.github/workflows/deploy.yml` 文件有更改
2. **输入提交信息**：
   ```
   Fix GitHub Actions deployment permissions and use official Pages actions
   ```
3. **点击 "Commit to main"**
4. **点击 "Push origin"**

### 2. 检查 GitHub 设置

1. **进入仓库 Settings** → **Actions** → **General**
2. **找到 "Workflow permissions"** 部分
3. **选择 "Read and write permissions"**
4. **勾选 "Allow GitHub Actions to create and approve pull requests"**（如果需要）
5. **点击 Save**

### 3. 重新运行工作流

推送后，GitHub Actions 会自动重新运行。或者：

1. 进入 **Actions** 标签
2. 点击失败的运行记录
3. 点击右上角的 **Re-run all jobs**

## 🔍 如果仍然失败

### 检查错误日志

1. 点击失败的运行记录
2. 展开失败的步骤（红色 X）
3. 查看错误信息

### 常见错误及解决方案

**错误：Permission denied**
- 解决：确保 Settings → Actions → General → Workflow permissions 设置为 "Read and write"

**错误：Build failed**
- 解决：检查构建日志，可能是依赖问题或代码错误

**错误：Deploy failed**
- 解决：确保 GitHub Pages 已启用（Settings → Pages → Source: GitHub Actions）
