# 图片优化指南

## 问题分析

根据 PageSpeed Insights 报告，以下图片需要优化：

1. **wallImage.png**: 2,508.7 KiB → 目标：< 200 KiB（节省 2,252.7 KiB）
2. **placeholder-icon.png**: 2,055.8 KiB → 目标：< 200 KiB（节省 1,885.2 KiB）
3. **upload-icon.png**: 148.4 KiB → 目标：< 20 KiB（节省 147.6 KiB），且尺寸需要调整为 66x72

## 优化步骤

### 方法 1：使用在线工具（推荐）

1. **访问图片压缩工具**：
   - Squoosh: https://squoosh.app/
   - TinyPNG: https://tinypng.com/
   - ImageOptim: https://imageoptim.com/online

2. **转换为 WebP 格式**：
   - 在 Squoosh 中打开图片
   - 选择 WebP 格式
   - 调整质量（建议 75-85）
   - 下载优化后的图片

3. **调整尺寸**（仅 upload-icon.png）：
   - 将 upload-icon.png 调整为 66x72 像素
   - 或使用 2x 分辨率：132x144 像素（用于高DPI屏幕）

### 方法 2：使用命令行工具

```bash
# 安装 imagemagick 或使用在线工具
# 转换为 WebP
convert wallImage.png -quality 80 wallImage.webp
convert placeholder-icon.png -quality 80 placeholder-icon.webp
convert upload-icon.png -resize 132x144 -quality 80 upload-icon.webp
```

### 方法 3：使用 Node.js 脚本

可以创建一个构建脚本自动优化图片。

## 文件替换

优化后，需要：

1. **替换原文件**：
   - `public/wallImage.png` → `public/wallImage.webp`
   - `public/placeholder-icon.png` → `public/placeholder-icon.webp`
   - `public/upload-icon.png` → `public/upload-icon.webp`（调整尺寸）

2. **更新代码引用**：
   - `src/ghibli-background.css`: 更新 `background-image: url('/wallImage.webp')`
   - `src/styles.css`: 更新 placeholder-icon 引用
   - `index.html`: 更新 upload-icon 引用，添加 `<picture>` 标签支持回退

## 预期效果

- **总大小**：从 4,712.9 KiB 降至约 420 KiB
- **预计节省**：4,285 KiB
- **加载速度**：显著提升，特别是移动端

## 注意事项

1. **WebP 浏览器支持**：现代浏览器都支持，但需要为旧浏览器提供 PNG 回退
2. **质量平衡**：不要过度压缩，保持视觉质量
3. **测试**：优化后测试图片显示是否正常
