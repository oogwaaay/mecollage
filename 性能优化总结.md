- **CLS**: 0.00, 得分 100 (权重 25%) - ✅ 很好

## 已实施的优化

### 1. ✅ 预加载 LCP 关键资源

**优化内容**：
- 预加载 `wallImage.webp`（背景图，影响 LCP）
- 预加载关键字体文件（Noto Sans SC weight 700，用于标题）

**代码位置**：`index.html` `<head>` 部分

```html
<!-- Preload critical resources for LCP optimization -->
<link rel="preload" href="/wallImage.webp" as="image" type="image/webp">
<link rel="preload" href="https://fonts.gstatic.com/s/notosanssc/v36/k3kXo84MPvpLmixcA63oeALhLq1mYj.woff2" as="font" type="font/woff2" crossorigin="anonymous">
```

**预期效果**：
- LCP 从 3,781 ms 降至 ~2,500 ms 以下
- FCP 从 3,189 ms 降至 ~1,800 ms 以下

### 2. ✅ 优化图片加载策略

**已优化**：
- 避免 PNG 预加载（只加载 WebP）
- 早期 WebP 检测
- 动态设置背景图片

### 3. ✅ 优化字体加载

**已优化**：
- DNS prefetch 和 preconnect
- 只加载使用的字体权重（400, 500, 600, 700）
- 使用 `font-display: swap`
- 预加载关键字体（weight 700，用于标题）

### 4. ✅ 优化 CSS 加载

**已优化**：
- 关键 CSS 立即加载
- 非关键 CSS 延迟加载（pages.css）
- 字体 CSS 延迟加载

### 5. ✅ 其他优化

- 对比度问题修复
- 主线程任务优化（requestAnimationFrame）
- 未合成的动画修复

## 进一步优化建议（可选）

### 1. 确保 WebP 文件已优化

**检查**：
- `wallImage.webp` 文件大小应该 < 200 KiB
- `placeholder-icon.webp` 文件大小应该 < 200 KiB
- `upload-icon.webp` 文件大小应该 < 20 KiB

**如果未优化**：
- 使用 Squoosh (https://squoosh.app/) 优化
- 参考 `WebP图片替换指南.md`

### 2. 内联关键 CSS（高级优化）

**如果评分仍低于 90**：
- 内联 `.ghibli-title` 样式（标题样式）
- 内联 `.ghibli-background` 样式（背景样式）
- 减少首次渲染阻塞

### 3. 使用 Service Worker 缓存（高级优化）

**可选**：
- 缓存关键资源（字体、背景图）
- 提升重复访问性能
