# 🚀 部署操作步骤（详细版）

## 情况说明

由于当前 PowerShell 环境中 Git 命令不可用，我为您准备了两种部署方式：

---

## 方式一：使用 GitHub Desktop（最简单，推荐）✨

### 步骤 1：打开 GitHub Desktop

1. 如果您已安装 GitHub Desktop：
   - 打开 GitHub Desktop
   - 点击 **File** → **Add Local Repository**
   - 选择文件夹：`C:\Users\Surface\Desktop\mecollage`
   - 点击 **Add repository**

2. 如果还没安装：
   - 下载：https://desktop.github.com/
   - 安装后按上述步骤操作

### 步骤 2：提交更改

1. 在 GitHub Desktop 中，您会看到左侧列出了所有更改的文件
2. 在底部输入提交信息：
   ```
   Add SEO optimizations, performance improvements, and GitHub Actions auto-deployment
   ```
3. 点击 **Commit to main**（或您的主分支名称）

### 步骤 3：推送到 GitHub

1. 点击右上角的 **Push origin** 按钮
2. 等待推送完成

### 步骤 4：启用 GitHub Actions

1. 在浏览器中打开您的 GitHub 仓库
2. 点击 **Settings** → **Pages**
3. 在 **Source** 中选择：**GitHub Actions**
4. 点击 **Save**

### 步骤 5：等待自动部署

1. 在仓库页面点击 **Actions** 标签
2. 您会看到 "Build and Deploy" 工作流正在运行
3. 等待 2-3 分钟，看到绿色 ✓ 表示成功
4. 网站自动更新完成！

## 方式二：使用 Git Bash 或命令行

### 步骤 1：打开 Git Bash

1. 右键点击 `mecollage` 文件夹
2. 选择 **Git Bash Here**（如果已安装 Git）

或者：

1. 打开 Git Bash
2. 使用 `cd` 命令进入项目目录：
   ```bash
   cd /c/Users/Surface/Desktop/mecollage
   ```

### 步骤 2：检查状态

```bash
git status
```

这会显示所有更改的文件。

### 步骤 3：添加所有更改

```bash

```

### 步骤 4：提交更改

```bash

```

### 步骤 5：推送到 GitHub

```bash

```

### 步骤 6：启用 GitHub Actions

1. 在浏览器中打开您的 GitHub 仓库
2. 点击 **Settings** → **Pages**
3. 在 **Source** 中选择：**GitHub Actions**
4. 点击 **Save**

### 步骤 7：等待自动部署

1. 在仓库页面点击 **Actions** 标签
2. 查看 "Build and Deploy" 工作流运行状态
3. 等待完成（约 2-3 分钟）

## 🆘 如果遇到问题

### Git 命令找不到？
- 安装 Git：https://git-scm.com/download/win
- 或者使用 GitHub Desktop（更简单）

### 推送被拒绝？
- 检查是否有未拉取的更改：`git pull` 先拉取
- 或者使用 `git push --force`（谨慎使用）

### GitHub Actions 没有运行？
- 检查 Settings → Actions → General
- 确保 "Workflow permissions" 设置为 "Read and write permissions"

### 网站没有更新？
- 等待 2-3 分钟
- 清除浏览器缓存
- 检查 Actions 标签中的部署状态
